## 技术要点

1. **并行 BVID→avid**：8 线程 `ThreadPoolExecutor`，982 个视频从 ~100s 降至 ~12s
2. **分流策略**：每个目标夹先查容量，`capacity = 999 - current`。容量不足时查 `OVERFLOW` 映射
3. **失效过滤**：`bvid_to_aid()` 返回 None 的视频自动跳过，不计入失败
4. **限流**：每批 move 后 `time.sleep(1.5)`，未触发 API 限流

## 教训

- `resource/clean` API 返回 `data:0` 是正常的，不表示"0 个清除"
- 大规模执行前必须 `--dry-run` 确认目标夹容量
- 溢出夹必须在移动前创建，不能边移边建
- 790 个"放弃区"视频全部 `view` API 返回 404，说明分类阶段已过期
